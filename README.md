# Student Management System (SMS)

## 📌 Project Description

This project is a simple Student Management System developed to practice and understand Git and GitHub workflows. It demonstrates how to manage project files, track changes, handle version control, and collaborate using Git.

The system includes basic front-end files such as HTML, CSS, and JavaScript, and will be updated step by step to simulate real-world software development scenarios.

## 📊 Git Commands Explanation

### 1. git log

This command shows the full commit history of the project. It includes details like commit ID, author, date, and commit message.

### 2. git log --oneline

This command shows a short version of the commit history. Each commit is displayed in a single line with a short commit ID and message.

### 3. git diff

This command shows the differences between changes in files. It helps to see what has been modified before committing.

## ⚙️ Git Commands Used

- git init → Initialize a new Git repository
- git add . → Add all files to staging area
- git commit -m "message" → Save changes with a message
- git status → Check current state of files
- git log → View full commit history
- git log --oneline → View short commit history
- git diff → Show changes between files
- git branch → List branches
- git checkout -b feature-login → Create and switch branch
- git merge → Merge branches
- git rm → Delete file from Git
- git restore / git checkout → Restore deleted files
- git push → Upload project to GitHub
- git pull → Get latest changes from GitHub

## ⚠️ Challenges Faced

During this project, I faced several challenges:

- Understanding how to restore deleted files using Git commands
- Confusion between file names (student.txt vs students.txt)
- Handling merge conflicts and resolving them manually
- Learning how branches work and how to switch between them
- Understanding the difference between local and remote repositories

These challenges helped me improve my understanding of Git and real-world development workflows.
